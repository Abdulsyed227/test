The problem

HTML/CSS

Follow the instruction below, try to match the final site to the screenshots for the follow devices: iPhone X, iPad, Desktop (1280 x 1080), Desktop (3440 x 1440).

You can get the images used in the example here.

  • Create a website that works from 3440px (ultrawide monitor) to 375px (mobile device)
  • The site should have a max width of 1440px and the site is centered on the page
  • The site supports background bleeding to the edge of the screen for different sections
  • Ultrawide Desktop, Standard Desktop, Tablet, Mobile
  • Don't need to worry about browser compatibility, just need to make sure your code works in the latest version of Chrome, Edge, Firefox, and Safari. You can use any experimental html/css feature you want.
  • Please try to solve this problem without using any css frameworks (no material-ui, bookstrap, foundation, tailwind, etc.). If you can use sass/less preprocessor, please do so and keep the source file in the project.

Javascript

  • Go to https://public.opendatasoft.com/explore/?sort=modified, find dataset for PM2.5 in the United States.
  • Generate a table for the first ten rows of the dataset.

Performance

  • The site needs to score high in the site speed test.

Bonus Point

  • Show the table as cards on mobile device.
